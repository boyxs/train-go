# AI 客服 — 产品需求文档（PRD）

## 1. 模块概述

AI 客服为小微书平台提供智能对话能力，用户通过聊天获取平台帮助、文章问答和个性化推荐。

本模块同时作为 **AI 全栈学习项目**，完整覆盖 Prompt Engineering、RAG、Function Calling、Streaming、对话记忆、Agent 编排六大方向。

### 待办功能

**P0 — 核心对话**
- ✅ 全局聊天气泡 + 弹层窗口（`ChatBubble.tsx` + `(main)/layout.tsx`）
- ✅ 独立聊天页（`/chat`，`(chat)/chat/page.tsx`）
- ✅ 流式输出（SSE 打字机效果，`api/chat.ts` + `ChatMessages.tsx`）
- ✅ 多轮对话上下文记忆（`chat.go` `maxHistoryRounds=20`，按 token 预算滑动窗口）
- ✅ 平台 FAQ 问答（Prompt Engineering，`systemPrompt` 角色设定 + 边界规则）
- ✅ 基于文章内容的知识问答（RAG，`searchArticles` → ES 向量检索 → 注入 prompt）

**P1 — Agent 能力**
- ⬜ 意图识别 + 路由分发（FAQ / RAG / Tool）
- ✅ Function Calling：文章搜索、收藏查询、热门推荐
- ✅ 对话历史列表 + 恢复上下文（`ChatSidebar.tsx` + `useConversations.ts`）
- ✅ 消息中的文章卡片（可点击跳转）

**P2 — 远期迭代**
- ⬜ 作者文章润色助手
- ⬜ 对话统计与热门问题分析
- ⬜ 用户反馈（点赞/点踩单条回复）

## 3. API 接口

### 对话管理

| Method | 路径 | 说明 | 认证 |
|--------|------|------|------|
| POST | `/chat/conversation/create` | 创建新对话 | JWT |
| POST | `/chat/conversation/list` | 对话历史列表 | JWT |
| POST | `/chat/conversation/delete` | 删除对话 | JWT |

### 消息

| Method | 路径 | 说明 | 认证 |
|--------|------|------|------|
| POST | `/chat/message/send` | 发送消息（返回 SSE 流） | JWT |
| POST | `/chat/message/list` | 获取对话消息列表 | JWT |
| POST | `/chat/message/stop` | 中断生成 | JWT |

**SSE 流格式：**

```
event: delta
data: {"content": "你", "type": "text"}

event: delta
data: {"content": "好", "type": "text"}

event: tool_call
data: {"name": "search_articles", "args": {"keyword": "Go并发"}}

event: tool_result
data: {"articles": [...]}

event: delta
data: {"content": "根据搜索结果...", "type": "text"}

event: done
data: {"usage": {"prompt_tokens": 520, "completion_tokens": 180}}
```

## 4. 用户故事

### P0 — 核心对话

| 角色 | 操作 | 价值 |
|------|------|------|
| 读者 | 点击气泡，输入"怎么发布文章" | 快速获取平台使用指引，无需翻找帮助文档 |
| 读者 | 提问"Go 并发有什么好文章" | AI 基于平台文章内容回答，附带原文链接（RAG） |
| 读者 | 连续追问 3 轮，每轮依赖上文 | 不用重复背景，AI 理解上下文连贯回答 |
| 读者 | 看到 AI 逐字输出回复 | 无需干等，流式体验自然流畅 |

### P1 — Agent 能力

| 角色 | 操作 | 价值 |
|------|------|------|
| 读者 | 问"推荐几篇热门文章" | AI 调用推荐工具，返回可点击的文章卡片 |
| 读者 | 问"我收藏了哪些文章" | AI 调用收藏接口，返回个人收藏列表 |
| 读者 | 点击历史对话恢复聊天 | 回顾过去的问答，继续追问 |

### P2 — 远期迭代

| 角色 | 操作 | 价值 |
|------|------|------|
| 作者 | 请 AI 润色文章摘要 | 提升文章质量，降低写作门槛 |
| 管理员 | 查看对话统计 | 了解用户常见问题，优化产品 |

## 5. 设计规范

沿用文章模块设计体系，AI 客服新增以下规范：

### 5.1 气泡与窗口

| 项 | 值 |
|---|---|
| 气泡位置 | 右下角，距底 24px，距右 24px |
| 气泡尺寸 | 48px 圆形，主色背景 |
| 弹层尺寸 | 宽 380px，高 560px，圆角 16px |
| 独立页布局 | 左栏 280px（历史列表）+ 右栏自适应（对话区） |

### 5.2 消息气泡

| 角色 | 背景色 | 对齐 | 圆角 |
|------|--------|------|------|
| 用户 | 主色（`#FF6B6B`），白字 | 右对齐 | 16px，右下 4px |
| AI | `#F3F4F6`，深色字 | 左对齐 | 16px，左下 4px |

### 5.3 特殊内容块

| 类型 | 样式 |
|------|------|
| 文章卡片 | 白底卡片，标题+摘要+阅读量，可点击跳转 |
| 代码块 | 深色背景，等宽字体，带复制按钮 |
| 工具调用中 | 灰色虚线框 + 加载动画 + "正在搜索文章..." |
| 流式光标 | 末尾闪烁竖线 `▊`，输出完成后消失 |

### 5.4 交互细节

- 发送：Enter 发送，Shift+Enter 换行
- 空消息不可发送，按钮置灰
- AI 生成中禁用发送，显示"停止生成"按钮
- 新消息自动滚动到底部，用户手动上滑时暂停自动滚动
- 气泡展开时有缩放 + 淡入动画，收起时反向

## 6. 核心流程

### 6.1 消息处理流程

```
用户输入
  │
  ▼
前端发送 POST /chat/message/send
  │
  ▼
后端鉴权 → 频率检查 → 构建 prompt
  │
  ├─ 系统提示词（角色设定 + 平台知识 + 工具定义）
  ├─ 历史消息（最近 N 轮，按 token 预算截断）
  └─ 当前用户消息
  │
  ▼
发送给 LLM（stream: true）
  │
  ├── 普通文本 ──→ SSE delta 事件 → 前端逐字渲染
  │
  ├── Tool Call ──→ SSE tool_call 事件
  │      │          → 前端显示"正在调用..."
  │      ▼
  │   执行工具（查文章/查收藏/推荐）
  │      │
  │      ▼
  │   工具结果注入上下文 → 继续生成
  │
  └── 完成 ──→ SSE done 事件
               → 保存完整消息到数据库
               → 前端移除光标，启用输入
```

### 6.2 RAG 检索流程

```
用户提问
  │
  ▼
Agent 判断需要文章知识
  │
  ▼
Tool Call: search_articles(query)
  │
  ▼
query → Embedding → 向量数据库 top-K 召回
  │
  ▼
召回文章片段注入 prompt：
  "以下是相关文章内容，请基于这些内容回答：
   [1] 《文章标题》: 内容片段...
   [2] 《文章标题》: 内容片段..."
  │
  ▼
LLM 基于检索内容生成回答 + 引用来源
```

## 7. 边界与约束

| 约束 | 规则 |
|------|------|
| 认证 | 仅登录用户，JWT 鉴权 |
| LLM Provider | DeepSeek / OpenAI，API Key 后端环境变量管理 |
| 传输协议 | SSE（Server-Sent Events），非 WebSocket |
| 单条消息上限 | 用户输入 ≤ 2000 字符 |
| 频率限制 | 单用户 10 条/分钟 |
| 上下文窗口 | 最近 20 轮对话，超出按时间截断 |
| 对话保留 | 30 天，过期自动清理 |
| 向量数据库 | 仅索引已发布文章（published_article） |
| 工具权限 | Tool 只读，不执行写操作（不替用户发文章、删文章） |
| 回复安全 | 系统提示词限定角色边界，拒绝无关话题 |
| 移动端 | 气泡保留，弹层全屏化，独立页隐藏左栏用 Drawer |

## 8. 验收标准

### P0 — 核心对话

```
Given  已登录用户在文章广场页
When   点击右下角 AI 气泡
Then   弹出聊天窗口，显示欢迎语"你好！有什么可以帮你？"

Given  聊天窗口已打开
When   输入"怎么发布文章"并发送
Then   AI 流式输出发布文章的操作步骤，有打字机效果

Given  用户已问过"Go并发"相关问题
When   追问"能详细说说 goroutine 吗"
Then   AI 理解上下文，基于前一轮回答继续展开

Given  平台有 Go 并发相关的已发布文章
When   用户问"有关于 Go 并发的文章吗"
Then   AI 检索文章内容回答，附带文章标题和链接

Given  AI 正在流式生成
When   网络中断
Then   保留已接收内容，显示错误提示和重试按钮

Given  AI 正在生成中
When   用户点击"停止生成"
Then   立即停止输出，已生成内容保留
```

### P1 — Agent 能力

```
Given  用户已登录且有收藏记录
When   问"我收藏了哪些文章"
Then   AI 调用收藏查询工具，返回收藏列表（文章卡片形式）

Given  用户问"推荐几篇热门文章"
When   AI 调用推荐工具
Then   返回热门文章卡片列表，点击可跳转到文章详情页

Given  用户有多个历史对话
When   点击"历史"
Then   按时间倒序展示对话列表，标题为对话首条消息摘要

Given  用户点击某个历史对话
When   对话加载完成
Then   显示完整对话记录，可以继续追问
```

## 9. 技术学习路线图

本模块按阶段递进，每个阶段对应一个 AI 核心技能：

| 阶段 | 学习目标 | 实现内容 | 预计产出 |
|------|----------|----------|----------|
| 1 | **Streaming + 基础对话** | SSE 流式接口、前端逐字渲染、消息存储 | 能聊天，有打字机效果 |
| 2 | **Prompt Engineering** | 系统提示词设计、角色边界、输出格式控制 | FAQ 问答准确、拒绝无关话题 |
| 3 | **对话记忆** | 多轮上下文管理、滑动窗口、token 预算截断 | 连贯多轮对话 |
| 4 | **RAG** | 文章向量化、Embedding、向量检索、结果注入 | 基于文章内容回答+引用来源 |
| 5 | **Function Calling** | Tool 定义、调用协议、结果回注 | AI 能查文章、查收藏、推荐 |
| 6 | **Agent 编排** | 意图识别、路由分发、多步骤任务编排 | 智能判断用 FAQ/RAG/Tool 回答 |

建议按 1→2→3→4→5→6 顺序开发，每阶段独立可用。
