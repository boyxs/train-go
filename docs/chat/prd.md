# AI 客服 — 产品需求文档（PRD）

## 1. 模块概述

AI 客服为小微书平台提供智能对话能力，用户通过聊天获取平台帮助、文章问答和个性化推荐。

本模块同时作为 **AI 全栈学习项目**，完整覆盖 Prompt Engineering、RAG、Function Calling、Streaming、对话记忆、Agent 编排六大方向。

### 待办功能

**P0 — 核心对话**
- ✅ 全局聊天气泡 + 弹层窗口（`ChatBubble.tsx` + `(main)/layout.tsx`）
- ✅ 独立聊天页（`/chat`，`(chat)/chat/page.tsx`）
- ✅ 流式输出（SSE 打字机效果，`api/chat.ts` + `ChatMessages.tsx`）
- ✅ 多轮对话上下文记忆（`chat.go` `maxHistoryRounds=20`，按 token 预算滑动窗口）
- ✅ 平台 FAQ 问答（Prompt Engineering，`systemPrompt` 角色设定 + 边界规则）
- ✅ 基于文章内容的知识问答（RAG，`searchArticles` → ES 向量检索 → 注入 prompt）

**P1 — Agent 能力**
- ⬜ 文章标签/分类体系（意图识别的前置依赖）
- ⬜ 意图识别 + 路由分发（FAQ / RAG / Tool，依赖标签体系）
- ✅ Function Calling：文章搜索、收藏查询、热门推荐
- ✅ 对话历史列表 + 恢复上下文（`ChatSidebar.tsx` + `useConversations.ts`）
- ✅ 消息中的文章卡片（可点击跳转）

**P2 — 远期迭代**
- ⬜ 作者文章润色助手
- ✅ AI 点击埋点 + 数据看板（`/ai/click` 埋点上报 + `/dashboard/ai` 数据看板，原型 `05-AI数据看板.png`）
- ⬜ 用户反馈（复制 + 点赞/点踩单条回复，原型 `06-消息反馈.png`）

### 对话管理

| Method | 路径 | 说明 | 认证 |
|--------|------|------|------|
| POST | `/chat/conversation/create` | 创建新对话 | JWT |
| POST | `/chat/conversation/list` | 对话历史列表 | JWT |
| POST | `/chat/conversation/delete` | 删除对话 | JWT |

### 消息

| Method | 路径 | 说明 | 认证 |
|--------|------|------|------|
| POST | `/chat/message/send` | 发送消息（返回 SSE 流） | JWT |
| POST | `/chat/message/list` | 获取对话消息列表 | JWT |
| POST | `/chat/message/stop` | 中断生成 | JWT |

### 消息反馈

| Method | 路径 | 说明 | 认证 |
|--------|------|------|------|
| POST | `/chat/message/feedback` | 设置消息反馈（1=赞, -1=踩, 0=取消） | JWT |

### P2 埋点 + 数据看板

| Method | 路径 | 说明 | 认证 |
|--------|------|------|------|
| POST | `/ai/click` | 记录用户点击 AI 推荐文章 | JWT |
| POST | `/ai/dashboard` | 获取 AI 引流数据看板 | JWT |

**POST /ai/click**

请求体：

```json
{
  "article_id": 123,
  "conversation_id": 456
}
```

响应：`{ "code": 0, "msg": "ok", "data": null }`

- `user_id` 从 JWT 提取，不由前端传
- 同用户 + 同文章 + 同对话只记一次（唯一索引去重，幂等）
- 前端 fire-and-forget，失败静默忽略不阻塞跳转

**POST /ai/dashboard**

请求体：无（固定最近 30 天）

响应：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "total_clicks": 342,
    "unique_users": 89,
    "unique_articles": 45,
    "avg_clicks_per_user": 3.8,
    "daily_trend": [
      { "date": "2026-04-01", "clicks": 12 },
      { "date": "2026-04-02", "clicks": 18 }
    ],
    "top_articles": [
      { "rank": 1, "article_id": 123, "title": "Go 并发编程", "clicks": 42, "unique_users": 18 }
    ]
  }
}
```

---

**SSE 流格式：**

```
event: delta
data: {"content": "你", "type": "text"}

event: delta
data: {"content": "好", "type": "text"}

event: tool_call
data: {"name": "search_articles", "args": {"keyword": "Go并发"}}

event: tool_result
data: {"articles": [...]}

event: delta
data: {"content": "根据搜索结果...", "type": "text"}

event: done
data: {"usage": {"prompt_tokens": 520, "completion_tokens": 180}}
```

## 4. 用户故事

### P0 — 核心对话

| 角色 | 操作 | 价值 |
|------|------|------|
| 读者 | 点击气泡，输入"怎么发布文章" | 快速获取平台使用指引，无需翻找帮助文档 |
| 读者 | 提问"Go 并发有什么好文章" | AI 基于平台文章内容回答，附带原文链接（RAG） |
| 读者 | 连续追问 3 轮，每轮依赖上文 | 不用重复背景，AI 理解上下文连贯回答 |
| 读者 | 看到 AI 逐字输出回复 | 无需干等，流式体验自然流畅 |

### P1 — Agent 能力

| 角色 | 操作 | 价值 |
|------|------|------|
| 读者 | 问"推荐几篇热门文章" | AI 调用推荐工具，返回可点击的文章卡片 |
| 读者 | 问"我收藏了哪些文章" | AI 调用收藏接口，返回个人收藏列表 |
| 读者 | 点击历史对话恢复聊天 | 回顾过去的问答，继续追问 |

### P2 — 远期迭代

| 角色 | 操作 | 价值 |
|------|------|------|
| 作者 | 请 AI 润色文章摘要 | 提升文章质量，降低写作门槛 |
| 管理员 | 查看 AI 点击数据看板 | 了解 AI 引流效果，证明 AI 功能价值 |
| 读者 | 问"推荐几篇热门文章"并点击文章卡片 | 行为被埋点记录，帮助平台分析用户偏好 |
| 读者 | 点击 AI 回复的复制按钮 | 快速复制回复内容，方便粘贴使用 |
| 读者 | 对 AI 回复点赞或点踩 | 表达满意度，帮助平台改进 AI 质量 |

## 5. 设计规范

沿用文章模块设计体系，AI 客服新增以下规范：

### 5.1 气泡与窗口

| 项 | 值 |
|---|---|
| 气泡位置 | 右下角，距底 24px，距右 24px |
| 气泡尺寸 | 48px 圆形，主色背景 |
| 弹层尺寸 | 宽 380px，高 560px，圆角 16px |
| 独立页布局 | 左栏 280px（历史列表）+ 右栏自适应（对话区） |

### 5.2 消息气泡

| 角色 | 背景色 | 对齐 | 圆角 |
|------|--------|------|------|
| 用户 | 主色（`#FF6B6B`），白字 | 右对齐 | 16px，右下 4px |
| AI | `#F3F4F6`，深色字 | 左对齐 | 16px，左下 4px |

### 5.3 特殊内容块

| 类型 | 样式 |
|------|------|
| 文章卡片 | 白底卡片，标题+摘要+阅读量，可点击跳转 |
| 代码块 | 深色背景，等宽字体，带复制按钮 |
| 工具调用中 | 灰色虚线框 + 加载动画 + "正在搜索文章..." |
| 流式光标 | 末尾闪烁竖线 `▊`，输出完成后消失 |
| 操作栏 | AI 气泡右下角，水平排列：复制 + 👍 + 👎，图标 14px，间距 8px |

### 5.4 交互细节

- 发送：Enter 发送，Shift+Enter 换行
- 空消息不可发送，按钮置灰
- AI 生成中禁用发送，显示"停止生成"按钮
- 新消息自动滚动到底部，用户手动上滑时暂停自动滚动
- 气泡展开时有缩放 + 淡入动画，收起时反向
- AI 回复操作栏：复制（灰色，点击 toast「已复制」）、👍（点击高亮主色）、👎（点击高亮红色）
- 操作栏仅 AI 消息显示，流式生成中隐藏，完成后出现
- 反馈支持取消（再次点击）和切换（赞↔踩），乐观更新

## 6. 核心流程

### 6.1 消息处理流程

```
用户输入
  │
  ▼
前端发送 POST /chat/message/send
  │
  ▼
后端鉴权 → 频率检查 → 构建 prompt
  │
  ├─ 系统提示词（角色设定 + 平台知识 + 工具定义）
  ├─ 历史消息（最近 N 轮，按 token 预算截断）
  └─ 当前用户消息
  │
  ▼
发送给 LLM（stream: true）
  │
  ├── 普通文本 ──→ SSE delta 事件 → 前端逐字渲染
  │
  ├── Tool Call ──→ SSE tool_call 事件
  │      │          → 前端显示"正在调用..."
  │      ▼
  │   执行工具（查文章/查收藏/推荐）
  │      │
  │      ▼
  │   工具结果注入上下文 → 继续生成
  │
  └── 完成 ──→ SSE done 事件
               → 保存完整消息到数据库
               → 前端移除光标，启用输入
```

### 6.2 RAG 检索流程

```
用户提问
  │
  ▼
Agent 判断需要文章知识
  │
  ▼
Tool Call: search_articles(query)
  │
  ▼
query → Embedding → 向量数据库 top-K 召回
  │
  ▼
召回文章片段注入 prompt：
  "以下是相关文章内容，请基于这些内容回答：
   [1] 《文章标题》: 内容片段...
   [2] 《文章标题》: 内容片段..."
  │
  ▼
LLM 基于检索内容生成回答 + 引用来源
```

## 7. 边界与约束

| 约束 | 规则 |
|------|------|
| 认证 | 仅登录用户，JWT 鉴权 |
| LLM Provider | DeepSeek / OpenAI，API Key 后端环境变量管理 |
| 传输协议 | SSE（Server-Sent Events），非 WebSocket |
| 单条消息上限 | 用户输入 ≤ 2000 字符 |
| 频率限制 | 单用户 10 条/分钟 |
| 上下文窗口 | 最近 20 轮对话，超出按时间截断 |
| 对话保留 | 30 天，过期自动清理 |
| 向量数据库 | 仅索引已发布文章（published_article） |
| 工具权限 | Tool 只读，不执行写操作（不替用户发文章、删文章） |
| 回复安全 | 系统提示词限定角色边界，拒绝无关话题 |
| 移动端 | 气泡保留，弹层全屏化，独立页隐藏左栏用 Drawer |

## 8. 验收标准

### P0 — 核心对话

```
Given  已登录用户在文章广场页
When   点击右下角 AI 气泡
Then   弹出聊天窗口，显示欢迎语"你好！有什么可以帮你？"

Given  聊天窗口已打开
When   输入"怎么发布文章"并发送
Then   AI 流式输出发布文章的操作步骤，有打字机效果

Given  用户已问过"Go并发"相关问题
When   追问"能详细说说 goroutine 吗"
Then   AI 理解上下文，基于前一轮回答继续展开

Given  平台有 Go 并发相关的已发布文章
When   用户问"有关于 Go 并发的文章吗"
Then   AI 检索文章内容回答，附带文章标题和链接

Given  AI 正在流式生成
When   网络中断
Then   保留已接收内容，显示错误提示和重试按钮

Given  AI 正在生成中
When   用户点击"停止生成"
Then   立即停止输出，已生成内容保留
```

### P1 — Agent 能力

```
Given  用户已登录且有收藏记录
When   问"我收藏了哪些文章"
Then   AI 调用收藏查询工具，返回收藏列表（文章卡片形式）

Given  用户问"推荐几篇热门文章"
When   AI 调用推荐工具
Then   返回热门文章卡片列表，点击可跳转到文章详情页

Given  用户有多个历史对话
When   点击"历史"
Then   按时间倒序展示对话列表，标题为对话首条消息摘要

Given  用户点击某个历史对话
When   对话加载完成
Then   显示完整对话记录，可以继续追问
```

### P2 — AI 点击埋点 + 数据看板

```
Given  用户在 AI 对话中看到文章卡片
When   点击文章卡片
Then   后台收到 POST /ai/click 请求，页面正常跳转到文章详情

Given  同一用户在同一对话中重复点击同一文章卡片
When   第二次点击
Then   不产生新的埋点记录（唯一索引去重），跳转正常

Given  POST /ai/click 请求失败（网络异常）
When   用户点击卡片
Then   仍正常跳转到文章详情，无错误提示

Given  已有点击数据
When   访问 /dashboard/ai
Then   页面显示四个概览卡片（总点击、独立用户、独立文章、人均点击）
       + 最近 30 天每日点击趋势折线图
       + 热门文章 Top 10 表格（排名、标题、点击次数、独立用户）

Given  暂无点击数据
When   访问 /dashboard/ai
Then   显示空状态提示："暂无数据，AI 对话中点击文章卡片后将自动记录"
```

**ROI Metrics**

| 指标 | 基线 | 目标 | 数据来源 | 窗口 |
|------|------|------|----------|------|
| AI 引流点击总次数 | 0 | >200次/周 | `ai_click_events COUNT(*)` | 上线后 4 周 |
| AI 引流 UV | 0 | >50人/周 | `COUNT(DISTINCT user_id)` | 上线后 4 周 |
| 人均 AI 点击次数 | 0 | >3次/周 | total / unique_users | 上线后 4 周 |
| AI 热门 Top10 与全站热门重合度 | — | >50% | hot-articles vs 全站热门榜 | 上线后 8 周 |

## 9. 技术学习路线图

本模块按阶段递进，每个阶段对应一个 AI 核心技能：

| 阶段 | 学习目标 | 实现内容 | 预计产出 |
|------|----------|----------|----------|
| 1 | **Streaming + 基础对话** | SSE 流式接口、前端逐字渲染、消息存储 | 能聊天，有打字机效果 |
| 2 | **Prompt Engineering** | 系统提示词设计、角色边界、输出格式控制 | FAQ 问答准确、拒绝无关话题 |
| 3 | **对话记忆** | 多轮上下文管理、滑动窗口、token 预算截断 | 连贯多轮对话 |
| 4 | **RAG** | 文章向量化、Embedding、向量检索、结果注入 | 基于文章内容回答+引用来源 |
| 5 | **Function Calling** | Tool 定义、调用协议、结果回注 | AI 能查文章、查收藏、推荐 |
| 6 | **Agent 编排** | 意图识别、路由分发、多步骤任务编排 | 智能判断用 FAQ/RAG/Tool 回答 |

建议按 1→2→3→4→5→6 顺序开发，每阶段独立可用。
