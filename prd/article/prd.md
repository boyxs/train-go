# 小微书 — 文章模块产品需求文档（PRD）

## 1. 模块概述

文章模块是小微书的核心功能，支持**作者写作管理**和**读者公开浏览**两个场景。

### 已完成功能

- 文章 CRUD（草稿/发布/撤回/删除）
- 双库分离（制作库 article + 线上库 published_article）
- 文章缓存（Redis Cache-Aside，TTL 30min + 抖动防雪崩）
- 读者公开文章列表 + 详情页
- 文章摘要（Content 前 128 字）
- 分页查询 + 全量列表
- 响应式适配（桌面 + 移动端）
- Modal.confirm 二次确认
- camelCase 统一字段命名
- Result JSON 统一响应格式

### 待办功能

- ~~文章摘要字段（abstract 独立字段）~~ ✅
- ~~点赞/收藏~~ ✅（互动模块已实现）
- ~~标签/分类 + 搜索~~ ✅（语义搜索已实现）
- 阅读量统计

## 3. API 接口

### 作者端（需登录）

| Method | 路径 | 说明 |
|--------|------|------|
| POST | `/article/edit` | 编辑文章（创建/更新草稿） |
| POST | `/article/publish` | 发布文章（制作库→线上库） |
| POST | `/article/withdraw` | 撤回文章（幂等） |
| POST | `/article/delete` | 永久删除文章 |
| POST | `/article/detail` | 文章详情（作者视角） |
| POST | `/article/page` | 文章分页 |
| POST | `/article/list` | 文章全量列表 |

### 读者端（公开）

| Method | 路径 | 说明 |
|--------|------|------|
| POST | `/article/reader/page` | 公开文章分页（含摘要） |
| POST | `/article/reader/detail` | 公开文章详情 |

## 4. 用户故事

### 作者端

| 优先级 | 故事 |
|--------|------|
| P0 | 作为作者，我希望创建草稿并保存，以便后续继续编辑 |
| P0 | 作为作者，我希望发布文章到线上，以便读者能看到 |
| P0 | 作为作者，我希望查看我的所有文章（分页），以便管理内容 |
| P0 | 作为作者，我希望编辑已有文章，以便修改内容 |
| P1 | 作为作者，我希望撤回已发布文章，以便从线上移除 |
| P1 | 作为作者，我希望永久删除文章，以便清理不需要的内容 |

### 读者端

| 优先级 | 故事 |
|--------|------|
| P0 | 作为读者，我希望浏览公开文章列表，以便发现内容 |
| P0 | 作为读者，我希望阅读文章全文，以便获取知识 |
| P0 | 作为未登录用户，我希望无需登录即可浏览，以便降低门槛 |
| P1 | 作为读者，我希望看到摘要预览，以便快速判断是否感兴趣 |
| P1 | 作为已登录用户，我希望从广场返回后台，以便切换模式 |

## 6. 边界与约束

| 约束 | 规则 |
|------|------|
| 数据来源 | 作者端查 `article` 表，读者端查 `published_article` 表 |
| 分页上限 | pageSize 最大 50，后端兜底 |
| 全量列表上限 | 1000 条 |
| 摘要长度 | Content 前 128 字符 |
| 排序 | id DESC（最新优先） |
| 响应式 | 桌面 768px 居中，移动端全宽 |
| 被撤回文章 | 读者端不可见 |
| 缓存 | 制作库 Cache-Aside，TTL 30min + rand(0~5min) |
| 安全 | Password `json:"-"` 不序列化，公开接口在 JWT 白名单 |
