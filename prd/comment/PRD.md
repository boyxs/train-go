## 1. 目标与场景

让读者在文章详情页就内容展开讨论，形成 UGC 互动沉淀。

- **核心场景**：读完文章 → 表达看法 → 与其他读者/作者互相回复 → 优质评论被点赞顶起
- **痛点**：当前文章详情只有点赞/收藏的"轻互动"，缺少"重表达"的讨论沉淀

## 2. 需求范围（已确认）

| 决策点 | 结论 |
|--------|------|
| 层级模型 | 存储**无限嵌套**（`pid` 自关联链）；**P0 展示扁平单层**（树形递归缩进留 P1） |
| 关联方式 | 通用 `biz` + `bizId`，P0 只接 `article`，视频/动态等后续扩展 |
| P0 功能 | 发表 / 查看列表 / 删除自己的 / 回复 + 评论点赞 + 排序(最新/最热) + @提及 + 敏感词过滤 + 加载更多 |
| 加载方式 | 加载更多（增量），非页码分页 |
| 权限 | 查看公开；发表/回复/点赞需登录；删除仅评论作者本人 |

## 3. 用户故事

| 优先级 | 角色 | 故事 |
|--------|------|------|
| P0 | 读者 | 在文章下发表评论，以便表达看法 |
| P0 | 读者 | 回复某条评论（可无限层），以便针对性讨论 |
| P0 | 读者 | 给评论点赞，以便认同优质内容 |
| P0 | 读者 | 按「最新/最热」切换排序，以便快速看到想看的 |
| P0 | 读者 | 删除自己发的评论，以便撤回不当言论 |
| P0 | 读者 | 分批「加载更多」，以便长列表不卡顿 |
| P0 | 平台 | 发布时拦截敏感词，以便内容合规 |
| P1 | 读者 | 举报不当评论 / 作者置顶优质评论 / 被回复时收到通知 |
| P2 | 读者 | 评论里插入表情、图片、富文本 |

## 4. 核心流程 + 异常分支

```
进入文章详情 → 评论区加载一级评论(默认「最热」，游标加载更多)
   ├─ 发表评论 → [未登录] 引导登录 ┊ [空/超长] 前端校验 ┊ [敏感词] 后端拦截 → 成功:乐观插入顶部
   ├─ 展开某楼回复 → 懒加载该 root 下子树(默认 3 条 +「展开更多回复」)
   ├─ 回复某条 → 输入框带「回复 @某人」→ 提交(校验同发表) → 成功:插入该节点 children
   ├─ 点赞/取消 → 乐观更新赞数 → [失败] 回滚+toast
   ├─ 切换排序 → 重置列表重新拉一级评论
   └─ 删除自己的 → 二次确认(一级评论有回复时提示「N 条回复将一并删除」)
        → 一级评论:整楼消失 ┊ 楼内回复:仅自身消失(子回复保留)
```

异常兜底：网络失败 toast 可重试；加载失败显示重试按钮；评论区整体加载失败不阻塞文章正文。

## 5. 页面/组件清单

评论功能**不是独立页面**，是嵌入文章详情页的组件树。

| 组件 | 挂载位置 | 认证 | 关键元素 |
|------|---------|------|---------|
| 文章详情页 `/article/read/:id` | 已存在 | 公开 | 评论区插入互动栏下方 |
| `CommentSection` 容器 | read.tsx 内 | 查看公开 | 标题计数、排序 Tab、顶部输入框、列表、加载更多 |
| `CommentItem` 评论项(递归) | 容器内 | — | 头像/昵称/时间、正文(含@高亮)、赞、回复、删除(仅作者)、子树 |
| `CommentEditor` 输入框 | 顶部 + 各回复处 | 写需登录 | 文本域、字数计数、提交、"回复 @X"前缀 |
| 空态 / 加载态 / 错误态 | 容器内 | — | "还没有评论" / 骨架屏 / 重试 |

## 6. 边界与约束

| 维度 | 约束 |
|------|------|
| 内容 | 1~500 字，前端实时计数 + 后端二次校验；禁止纯空白 |
| 嵌套 | 存储无限(`pid` 链)；**P0 展示扁平单层**（树形递归缩进 + 深层收敛留 P1） |
| 权限 | 查看公开；发表/回复/点赞需登录；删除仅评论作者本人（作者删他人 = P1） |
| 排序 | 仅对**一级评论**排序（最热=赞数倒序，最新=时间倒序）；楼内回复恒按时间正序 |
| 频率 | 发表/回复接入限流（复用 `pkg/ratelimit`），防刷 |
| 性能 | 列表查询 `Select` 排除大字段；热评/计数走缓存；游标分页避免深翻页 |
| 删除 | 软删除；删一级评论**整楼级联**（前置确认提示子回复数）；删楼内回复仅自身，子回复保留 |
| 多端 | 移动端缩进收敛，横向不溢出 |
| 合规 | 敏感词发布同步校验（实现选型见架构文档） |

## 7. 验收标准（Given-When-Then）

- **发表**：Given 已登录读者 When 提交 1~500 字正常内容 Then 评论出现在列表顶部且计数+1
- **敏感词**：Given 内容含敏感词 When 提交 Then 后端拒绝，前端提示"内容含敏感词"，不入库
- **回复**：Given 任意评论 When 点击回复并提交 Then 作为楼内回复扁平展示（P0 单层；@提及/树形缩进 P1）
- **点赞**：Given 任意评论 When 点赞 Then 赞数+1 且按钮高亮；再次点击取消 Then 赞数-1
- **排序**：Given 评论列表 When 切到"最热" Then 一级评论按赞数倒序；切"最新"按时间倒序
- **删除**：Given 自己的一级评论（有 N 条回复） When 点删除 Then 确认框提示"删除后，N 条回复也将一并删除"，确认后整楼消失且计数同步减；Given 自己的楼内回复 When 确认删除 Then 仅自身消失，其子回复保留；**他人评论无删除按钮**
- **未登录**：Given 未登录 When 点发表/回复/点赞 Then 引导登录，不报错
