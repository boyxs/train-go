# Feed 关注流 — 产品需求文档（PRD）

> 状态：待确认 | 会话决策：入口=广场页双 Tab · 翻页=游标+无限滚动 · 卡片=广场卡片+互动计数+标签

## 1. 模块概述

关注流（Follow Feed）：把「我关注的作者新发布的文章」按发布时间倒序聚合成一条信息流。现有文章广场页（`/feed`）升级为**双 Tab**——「关注」（本模块，需登录）+「发现」（现有广场列表，保持公开与页码分页不动）。

依托已上线能力：relation（关注关系 + `relation_events` 已按 follower_id 分区）、interaction（点赞/收藏计数）、comment（评论计数）、tag（文章标签）、core BFF 聚合模式。

### 本期范围（P0 + P1）

- ⬜ 广场页双 Tab 改造（关注 | 发现），Tab 状态写 URL
- ⬜ 关注流列表：游标 + 无限滚动，卡片含互动计数与标签
- ⬜ 空态（未关注任何人）/ 未登录引导
- ⬜ 新内容提示（P1：停留时有新文章 → 顶部提示条）

### 不做（后续迭代 P2）

- 推荐流（算法分发）、卡片上直接点赞/收藏、Header 红点未读数、下拉刷新手势

## 3. API 接口

### P0

| Method | 路径 | 说明 | 认证 |
|--------|------|------|------|
| POST | `/feed/list` | 关注流游标分页 | JWT |

请求 / 响应（camelCase，统一 `{code, msg, data}` 包裹）：

```jsonc
// 请求
{ "cursor": 1770000000000, "limit": 10 }   // cursor 缺省 = 拉最新一页；limit 默认 10，最大 20

// data
{
  "list": [
    {
      "articleId": 1001,
      "title": "Go 并发编程入门",
      "abstract": "goroutine 是 Go 语言最核心的并发原语...",
      "author": { "id": 7, "nickname": "阿哲" },
      "publishedAt": 1770000000000,        // Unix 毫秒
      "likeCnt": 128, "collectCnt": 32, "commentCnt": 24,
      "tags": [ { "id": 3, "name": "Golang" }, { "id": 9, "name": "并发" } ]
    }
  ],
  "nextCursor": 1769990000000,             // 下一页游标；hasMore=false 时忽略
  "hasMore": true
}
```

### P1

| Method | 路径 | 说明 | 认证 |
|--------|------|------|------|
| POST | `/feed/new-count` | 自某游标以来的新文章数（提示条用） | JWT |

> 接口由 core BFF 暴露并聚合昵称 / 互动数 / 评论数 / 标签（对齐现有 `GRPCCommentService` / `GRPCRelationService` 聚合模式）；feed 服务本体、读写扩散策略等见 architect 产出。

## 4. 用户故事

### P0 — 核心路径

| 角色 | 操作 | 价值 |
|------|------|------|
| 已登录用户 | 在「关注」Tab 按时间倒序浏览我关注作者的新文章 | 不错过关注的人的更新 |
| 已登录用户 | 触底自动加载更早内容（无限滚动） | 连续浏览不中断 |
| 已登录用户 | 点卡片新标签打开文章阅读页 | 顺畅进入阅读 |
| 已登录用户 | 卡片上看到点赞/收藏/评论数与标签 | 快速判断内容热度与主题 |
| 未关注任何人的用户 | 看到空态引导并一键切到发现页 | 建立关注列表 |
| 未登录用户 | 点「关注」Tab 看到登录引导（可去登录/先逛发现） | 明确下一步，不吃闭门羹 |
| 作者 | 发布文章后粉丝的关注流能看到（分钟级内） | 内容触达粉丝 |
| 作者 | 撤回文章后关注流不再展示 | 内容可控 |

### P1 — 首版应有

| 角色 | 操作 | 价值 |
|------|------|------|
| 已登录用户 | 停留时有新文章 → 顶部「有 N 篇新文章」提示，点击回顶刷新 | 感知实时性 |
| 已登录用户 | 取关某作者后，刷新关注流不再出现其文章 | 流内容与关注关系一致 |
| 已登录用户 | 列表加载失败看到错误提示并可重试 | 失败可恢复 |

### P2 — 后续迭代

| 角色 | 操作 | 价值 |
|------|------|------|
| 用户 | 「发现」升级推荐流（算法分发） | 千人千面 |
| 用户 | 卡片上直接点赞/收藏 | 减少跳转 |
| 用户 | Header 全局红点提示关注流新内容 | 全站召回 |

## 5. 设计规范

全局 token 见 `webook-fe/.claude/rules/design-tokens.md`（主色 `#0D9488`），本模块差异点：

| 元素 | 规范 |
|------|------|
| 双 Tab | 居中，间距 40；激活：15px/700 主色 + 下划线条 28×3 圆角 2；未激活：15px/500 `#6B7280` |
| 标签 chip | 对齐 tag 模块：圆角 12（pill）、底 `#0D948815`、文字 `#0D9488` 12px/500、内边距 4×10 |
| 互动计数 | lucide `heart / star / message-circle` 14px + 数字 12px/500，同 `#9CA3AF` |
| 相对时间 | <1h N 分钟前 · <24h N 小时前 · 昨天 HH:mm · <7d N 天前 · 更早 YYYY-MM-DD |
| 加载指示 | `loader-circle` 16px + 「加载中...」13px `#9CA3AF`；到底显示「没有更多了」 |
| 新内容提示条 | 主色胶囊（圆角 9999，内边距 8×16），白字 13px/600 + `arrow-up` 14px，悬浮列表顶部 |
| 空态 / 引导卡 | 白卡圆角 12，图标 40px `#D1D5DB`，主文案 15px/600 `#6B7280`，副文案 13px `#9CA3AF`，主按钮主色 |

## 6. 边界与约束

| 约束 | 规则 |
|------|------|
| 内容范围 | 仅「已发布」文章；撤回后不可见（读取时以线上库为准兜底过滤） |
| 时效 | 发布 → 粉丝关注流可见 P95 ≤ 60s（异步扩散，容忍分钟级） |
| 取关 | 取关后刷新不再出现该作者文章（读时过滤兜底） |
| 拉黑 | 拉黑自动解除双向关注（relation 已有级联）→ 关注流自然不含对方内容 |
| 分页 | 游标 = 上一批最后一条 `publishedAt`；`limit` 默认 10、最大 20，后端兜底 |
| 回溯上限 | 每用户收件箱保留最近 2000 条；更早显示「没有更多了」 |
| URL 状态 | Tab 写 URL：`?tab=follow / discover`（默认 discover 时参数省略）；发现 Tab 保留现有页码参数；关注 Tab 游标**不**进 URL |
| 排序 | `publishedAt DESC`，同毫秒以 `articleId DESC` 决胜 |
| 错误处理 | 列表加载失败必须有可感知提示 + 重试入口（前端错误处理零容忍） |
| 响应式 | ≥768px 内容区 768 居中；<768px 全宽、卡片内边距 16 |

## 7. 验收标准

```
Given 已登录且关注的作者发布了新文章（1 分钟前）
When  打开 /feed?tab=follow
Then  该文章出现在列表首位，卡片含标题/摘要/作者/相对时间/点赞收藏评论数/标签

Given 关注流已加载第一页且 hasMore=true
When  滚动触底
Then  自动携带 nextCursor 请求下一页并追加渲染，期间显示"加载中..."

Given hasMore=false
When  滚动触底
Then  显示"没有更多了"，不再发起请求

Given 未登录
When  访问 /feed?tab=follow
Then  显示登录引导卡（去登录 / 先逛逛发现），不发生整页跳转登录

Given 已登录但未关注任何人
When  打开关注 Tab
Then  显示空态卡（还没有关注任何人 + 去发现按钮），点击切到发现 Tab

Given 已登录用户停留在关注 Tab（P1）
When  期间关注的作者发布了 3 篇新文章
Then  列表顶部出现"有 3 篇新文章"提示条，点击回到顶部并刷新出新内容

Given 用户取关了作者 A
When  刷新关注流
Then  列表不再出现 A 的任何文章

Given 作者撤回了文章 X
When  任意粉丝刷新关注流
Then  X 不出现在列表中

Given /feed/list 请求失败
When  首屏或翻页加载
Then  显示错误提示与重试按钮，点击重试恢复
```
